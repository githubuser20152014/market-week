Shared Components (used by both flows)
New in this redesign
Smart Price Resolver verify_prices.py
  • Step 1 — Date offset: tries ±1 trading day on secondaries before flagging
  • Step 2 — Majority vote: if 2 of 3 sources agree within tolerance → auto-resolve, mark outlier
  • Step 3 — Hard halt: only if all 3 sources disagree · prints diagnostic table, not stack trace
  • Writes output/price_resolution_YYYY-MM-DD.txt on every run
Live Econ Calendar Fetch fetch_econ_calendar.py
  • Primary: FRED releases/dates API (free, allowlist of 14 releases)
  • Fallback: Finnhub /calendar/economic (if FRED key absent)
  • Past week: fetches actuals from FRED series observations
  • Upcoming week: date + label + importance only (no actuals yet)
  • Writes fixture · falls back to nearest existing fixture with warning
